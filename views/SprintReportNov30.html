Project Status Report

Sprint 4 - November 30, 2017

All Sprint Reports: http://project-ninja.herokuapp.com/progress

Completed stories in sprint 4:

  • API functionality returns valid JSON (4 story points)
  • Fix HTML tags and attributes to match the test cases (3 story points)
  • Backend services for RSVP, including databases (3 story points)
  • Validation for new event inputs (2 story points)
  • Website Analytics: Add donation link (2 story points)

For the fourth sprint, we continued to use Skype for our team meetings and Trello for our project management processes. Since we had two weeks in this sprint instead of one week and couldn’t see each other during class times, we tried to collaborate over Skype and Messenger to update each other about the latest status. Through our meeting, we decided to work on the stories listed above with a total of 14 story points, about the same amount of points as these of previous weeks.

While we were continued to working on bug fixes, we also worked on created new features. Thus, the real challenges in this sprint were creating the API and understanding how to use Google Analytics, which was part of enabling donation functionality. About API development, we had to code in a way that the responses are structured exactly like the example in the Git test repository. This restriction actually changed the way we initially wanted to implement, but at the same time, it allowed us to understand what exactly we should achieve. Setting up Google Analytics was not as hard as we thought, but it did take a good amount of our time to make it available. We will explore more about all the possible features of Google Analytics as we obtain more traffic, (i.e. having a good amount of data).

Burn-down Chart:

Current product backlog: For the last sprint we have the below items to complete.  However, more items may be added.

  1. Code security review for SQL injection, CSRF, or XSS vulnerabilities (5 story points)
  2. Code structure review for conformity with 12factor app principles (4 story points)
  3. API is searchable (3 story points)
  4. Obtain traffic reports from Google Analytics  (2 story points)
  5. New Event bug: last test case (1 story point)

Given the test results (which is attached at the end of this  sprint report), we are in a good position in terms of the overall development progress. Since we only have one more sprint, we will finish up the remaining 5 failed items in the current test report, corresponding to items 1, 2, 3, 5 in the current product backlog. In addition, we will analyze the traffic using Google Analytics and answer the questions about effectiveness of “Donation” vs “Support.”
