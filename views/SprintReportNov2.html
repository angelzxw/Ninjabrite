Project Status Report

Sprint 1 - November 2, 2017

Completed stories in sprint 1:

  • View all the events: Users should be able to list all the events from the home page (with the title and date/time of each event included).  (5 story points)
  • View the "About" page: Users should be able to navigate to the “About” page from the home page. The “About” page should contain an image for each team member together with their class nicknames.  (4 story points)
  • Create events: Users should be able to create new events by filling out the required information in an online form.  (2 story points)
  • View the details of an event: Users should be able to see the details for each event (title, date/time, location, image, attendees).  (1 story point)
  • ‘New event’ page form field validations with appropriate errors (1 story point)

After some minor technical issues, we were able to use Skype to get our team meetings up and running.  During the initial sprint planning meeting, we decided to use Trello to help us with project management and it helped us collaboratively decide on what stories we intended to complete during this sprint. We decided to first focus on creating the basic structure of each page in the app so that we would have a foundation on which to build in future sprints.

By the end of this sprint, we were able to achieve all the software implementations associated with these five stories that we prioritized among many others.  To make the website available, the team was split into two groups working on the front-end design and back-end configuration.  Specifically, the front-end group created new web pages, built the general layout template, and designed each available page involving the use of a lot of CSS.  As for the backend implementations, it included successfully setting up the server to host the web application, creating modulars that enable database queries (i.e. completing the event models), and modifying controllers according to the changes made in associated html pages.  The challenges during the development process included estimating the difficulty of stories, the time required for finishing individual implementation tasks, and the level of collaboration between the frontend and the backend developers.  However, we managed all these challenges by doing coding bootcamp and practising pair programming.  Overall, we maintained a good, steady pace working towards the goals of this sprint, and we did it.
