Project Status Report

Sprint 2 - November 9, 2017

Completed stories in sprint 2:

  • RSVP to an event: Users should be able to RSVP to an event using a form on the “Event Details” page with specified quantity and valid email addresses.  (4 story points)
  • Team logo on home page: Users should be able to see the team logo on the home page.  (4 story points)
  • “Event Details” page email address validation: Users should be able to RSVP to events using only a valid yale.edu email address. However, the form should accept any weird capitalizations.  (1 story point)
  • Bug fix: “New Event” page form to have field validations with appropriate errors.  (1 story point)
  • Bug fix: “New Event” page form to have drop-down menus for date and time fields.  (1 story point)

We continued to use Skype for our team meetings and Trello for our project management processes, as these tools allowed us to work efficiently in the first sprint.  For our second sprint, we decided to focus on implementing the new RSVP functionality and to perform bug fixes for some of the date and time values and field validation issues on the “New Event” page that were identified. A new team logo was designed for use on the home page. For this sprint, the challenges included syncing the Heroku app with Heroku PostgreSQL database services, exploring the APIs of the pg package on Node.js, and designing efficient database schemas. However, after spending hours of effort studying the documentation of these tools, we managed to solve these challenges and implement dynamic functionalities in our app.

Given what we have built this far, we still have a couple of bug fixes that we need to do in the next sprint. These bug fixes are associated with values displayed in the home page and About page.  While we have not yet determined the final contents of the next sprint, we will be focusing on items 1, 2, 3 and 4.
