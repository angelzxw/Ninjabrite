Project Status Report

Sprint 3 - November 16, 2017

All Sprint Reports: http://project-ninja.herokuapp.com/progress

Completed stories in sprint 3:

  • Bug fix: Correction for testing: “Home page should show the correct date and time fields for events”.  (2 story points)
  • Bug fix: Correction for testing: “The home page should have the team logo.” (1 story point)
  • Bug fix: Correction for testing: “About page should show class nicknames instead of real names of team members.” (1 story point)
  • Bug fix: Correction for testing: “In new event page, every form field should be filled out with valid input.” (2 story point)
  • Bug fix: Correction for testing: The ids used in the code should match those required by the test suite. (5 story points)

For the third sprint, we continued to use Skype for our team meetings and Trello for our project management processes. We managed the challenges by keeping constant communication with team members and practising pair programming.

Before this sprint, we had completed many functional requirements and designed the layout of the website in a way that we thought would be user-friendly. However, we now realized that in order to fully pass the test program, we should modify the web app code according to the  prose explanation of the test requirements. By analyzing the outputs of the test program, we found that most issues were caused by the lack of assigning identification or class names (e.g. need to add id=”logo”, class=”event”, etc ) or by the wrong definition of the routing paths to different pages. Thus, we decided to focus on fixing the issues mentioned above, which occurred in the Home, About, Events, Event Details, and New Event pages. The main challenges for this sprint were reviewing and understanding the code that were written by other team members, and then modifying the code without causing new bugs. To overcome the challenges, we had good communication among team members.

Overall, we maintained a good, steady pace working towards the goals of this sprint. As a result, we managed to complete the stories that we set for this week. To be noted, with the exception of the About page bug, fixing the above bugs, however, would not have any direct impact (visual or functional) to the product that we have built thus far.

Upon successful completion of this sprint, we will have very few requirements and potential bugs to implement. We are trying to complete as many stories as possible before the project deadline, since we may have other requirements to complete in the event something changes or goes wrong.
